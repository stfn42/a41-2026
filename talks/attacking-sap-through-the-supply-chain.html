Attacking SAP Through The Supply Chain
SPEAKER: Andreas Wiegenstein DURATION: 30:19
SUMMARY

SAP operates as a massive, 86-million-line proprietary network inside corporate perimeters, controlling the backbones of global logistics, automotive production, and finance. Despite standard perimeter defenses, SAP environments are uniquely vulnerable through their supply chain. The speaker highlights that relying on firewalls and basic access controls creates a false sense of security, primarily because third-party add-ons provide a direct path into the heart of the system. They use real-world incidents to demonstrate the stakes: an SAP component compromise at Jaguar Land Rover cost an estimated ┬г2 billion and halted production for six weeks, while a ransomware attack on an SAP subsystem forced the Stoli Group into bankruptcy.

The core of the threat lies in Advanced Business Application Programming (ABAP), the language running the SAP environment. ABAP programs operate with systemic dominanceтАФthey can execute arbitrary SQL queries, run operating system commands on the server, and hijack the machines of logged-in clients. Worse, ABAP code lives directly inside the SAP database and contains the native commands necessary to modify, compile, and hide its own source code. Attackers distribute malicious ABAP code via SAP transport files. These proprietary, black-box packages can trigger auto-execution the moment they are imported, meaning malware is actively running before any post-deployment security scanner can even inspect it.

Once a malicious transport executes, the payload becomes a permanent fixture of the system. Because ABAP code resides in the database, the malware automatically replicates into every standard database backup. They explain that if an attacker designs the payload to stay dormant longer than the organization's maximum feasible data rollback window, the malware becomes irrecoverable; restoring from a backup simply reinstalls the threat. Their proof-of-concept malware demonstrated this devastating capability, infecting 900 SAP programs in 1.5 seconds, reprogramming developer tools to render itself invisible, and successfully issuing commands through connected clients to physical operational technology (OT) devices, such as Siemens PLCs.

The supply chain blind spot is severe. In an analysis of 51 German enterprises, they found that 98 percent (50 companies) were running highly vulnerable, unpatched third-party SAP applications without knowing it. These vulnerabilities persist because companies lack basic software bill of materials (SBOM) catalogs for their SAP add-ons and configure code scanners incorrectly, failing to target third-party namespaces (which begin with a slash rather than the custom Y or Z prefixes). To defend against transport-based attacks, they recommend testing all updates in isolated sandboxes, demanding explicit deletion transports from vendors to purge code cleanly, and recognizing that official SAP certification guarantees only API compatibility, not security.

KEY TAKEAWAYS
  • SAP environments run on ABAP, a language that grants programs unrestricted authority to execute operating system commands, rewrite their own source code, and hijack the devices of connected clients.
  • Malicious SAP transport files can auto-execute proprietary code immediately upon import, bypassing static analysis and compromising systems before security tools can even run.
  • Because ABAP code is stored directly inside the SAP database, malware automatically infects all subsequent system backups, making disaster recovery impossible if the payload remains dormant past the rollback window.
  • Enterprise code scanners routinely miss third-party SAP vulnerabilities because administrators fail to explicitly configure scans for proprietary vendor namespaces.
  • Official SAP certification only validates API and interface compatibility, providing zero assurance that a third-party application is actually secure.
  • Organizations must isolate third-party software testing in dedicated sandboxes and demand explicit deletion transports from vendors to ensure dangerous code can be cleanly removed from the database.
